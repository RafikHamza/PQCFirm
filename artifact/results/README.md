# PQCFirm Results Directory

This directory holds the result files used by the verification harness. Some files are cached evidence bundled with the artifact; others are regenerated when `verify_all.py` runs.


## Top-level run summaries

The wrapper scripts also write compact JSON summaries here:

- `run_replication_summary.json`: generated by `run_replication.bat` or `run_replication.sh` after the full verification harness finishes.
- `hardware_capture_summary.json`: generated by `run_hardware.bat`, `run_hardware.sh`, or direct calls to `run_hardware_capture.py`.
- `latest_run_summary.json` and `latest_hardware_run.json`: convenience copies of the most recent summaries.
- `run_replication_console.log` and `hardware_capture_console.log`: copied terminal output from the wrapper scripts.
- `user_supplied_run_replication_console_20260629.txt` and `user_supplied_hardware_capture_console_20260629.txt`: user-supplied Windows run logs used to patch and cross-check this package.

Use these summaries for quick checks, and use the specific result files below for detailed evidence.

## Main result files

- `taxonomy_statistics.json`: taxonomy distribution over 1,043 mined commits.
- `annotation_consistency/`: four-pass annotation consistency files for the 200-commit sample.
- `pqcfirm_embedded_findings.json`: PQCFirm findings for the ESP32-S3 benchmark.
- `loc_count.json`: reproducible LOC count for the packaged application-level ESP32-S3 benchmark.
- `ground_truth_annotations.csv`: ESP32-S3 finding review used for the 67/68 precision result.
- `mbedtls_findings.json`: cached raw Mbed TLS finding list.
- `mbedtls_audit.csv`: raw Mbed TLS large-library stress-test audit.
- `mbedtls_audit_status.json`: raw Mbed TLS assisted triage status.
- `mbedtls_filtered/`: production-focused filtered Mbed TLS view.
- `paired_trials.csv`: 100 hardware paired-trial rows with `source=hardware_serial`.
- `failure_reproduction_status.json`: hardware crash/success status.
- `differential_status.json`: differential-testing status.
- `claim_matrix.json`: machine-readable claim support matrix.

## Notes on interpretation

The Mbed TLS files are assisted triage evidence. The raw audit shows the burden of running a lightweight scanner over a large general-purpose cryptographic library. The filtered view removes obvious non-production and cleanup/symmetric-hash noise before sampling candidates. Neither file is an independent human precision audit.

The curated seeded corpus files are rule-coverage evidence. They should not be interpreted as production recall.

## Regeneration

From the repository root:

```bash
bash run_replication.sh
```

On Windows:

```bat
run_replication.bat
```

To rerun only the harness after setup:

```bash
cd artifact
python verify_all.py
```

Hardware logs can be regenerated from the repository root with an ESP32-S3 attached:

```bash
python run_hardware_capture.py --port COM8 --benchmark-timeout 300
```

Use the serial port shown by your operating system instead of `COM8` when needed.
