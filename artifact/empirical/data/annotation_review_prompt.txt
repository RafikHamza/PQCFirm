You are an expert in post-quantum cryptography (PQC) migration for embedded firmware. Your task is to review a commit message and classify whether the commit represents a PQC-migration-relevant defect under one of the nine taxonomy categories (D1–D9) defined in the PQCFirm annotation codebook.

## Codebook Summary

| ID | Category |
|----|----------|
| D1 | Hardcoded Key / Buffer Sizes |
| D2 | Classical Algorithm Selection (no PQC branch) |
| D3 | Certificate / Public-Key Format Assumptions |
| D4 | Unchecked Return Values of Cryptographic Functions |
| D5 | Hardcoded Stack Size / Task Stack Allocation |
| D6 | Conditional Compilation for Feature Selection |
| D7 | Hardcoded Connection / Timeout Parameters |
| D8 | Cryptographic API Misuse / Deprecated API |
| D9 | Memory Safety Issues in Crypto Code |

## Classification Rules

1. If the commit is not related to PQC migration, output: D0
2. If the commit is related to PQC migration, choose the single most relevant D1–D9 category.
3. If multiple categories apply, choose the primary one (first encountered in the commit purpose).

## Output Format

For each commit, produce a single JSON object with these fields:
- commit_id: the commit SHA or identifier
- original_label: the initial keyword-assisted label (may be D0–D9)
- assisted_label: your LLM-assisted label (D0–D9)
- agreement: true if assisted_label matches original_label, false otherwise
- confidence: one of "high", "medium", "low"
- rationale: 1-2 sentence explanation for your classification
- review_method: "llm_assisted_codebook_review"

## Important Notes

- This is an LLM-assisted codebook review, NOT independent human double-coding.
- The goal is to validate the reproducibility of the keyword-assisted labeling methodology.
- Be conservative: if uncertain, note the uncertainty in the rationale rather than forcing a classification.
